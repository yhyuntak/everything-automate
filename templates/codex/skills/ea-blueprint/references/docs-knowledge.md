# Docs Knowledge

Use when the blueprint is about documentation, knowledge capture, indexes, or doc maintenance rules.

Design forces:
- Keep the source of truth clear.
- Match the doc shape to the reader and use case.
- Prevent drift between docs and the system they describe.

Risk hints:
- Stale or duplicate docs.
- Too much detail for the intended reader.
- Missing ownership or update path.

Output focus:
- Doc structure, section shape, and update rules.
- Source links and ownership cues.
- What needs to stay aligned as the system changes.
