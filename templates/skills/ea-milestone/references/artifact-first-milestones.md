# Artifact-First Milestones

Milestone names should describe finished outputs, not vague activity.

Good examples:

- repository investigation summary complete
- guide document v1 complete
- frontend guide design note accepted
- install flow validation notes complete

Weak examples:

- investigate the repo
- work on setup
- keep improving
- continue implementation

Each milestone should answer:

- what artifact exists when this milestone is done?
- what does that artifact unlock next?
- what is intentionally not included yet?
