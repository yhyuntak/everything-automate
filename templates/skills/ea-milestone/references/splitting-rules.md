# Milestone Splitting Rules

Use milestone splitting when the locked North Star is too large for one clean brainstorming and planning pass.

Split when one or more of these are true:

- there is clear early-vs-later order
- later work depends on an earlier artifact
- the goal changes artifact kind across stages
- one design pass would otherwise mix roadmap, document, and implementation design
- the user needs explicit handoff points

Do not split just to create more stages.

Prefer the smallest roadmap that preserves:

- correct ordering
- clear handoff artifacts
- one current milestone recommendation

One milestone is valid when the goal is already small enough.
