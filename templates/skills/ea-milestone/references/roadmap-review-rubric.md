# Roadmap Review Rubric

Use this rubric before locking a milestone roadmap.

Pass when:

- the parent goal is clearly preserved
- the roadmap is output-first
- the order is defensible
- one current milestone recommendation is explicit
- dependencies are clear enough for Blueprint
- later concerns are not mixed into the current milestone

Fail when:

- milestone names are action-only
- no current milestone is recommended
- the roadmap quietly changes the North Star
- ordering is arbitrary
- a later milestone depends on an earlier missing artifact
- blueprint would still need to redesign the roadmap itself
