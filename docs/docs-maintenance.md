# Docs Maintenance

This file is the quick map for keeping the docs fresh.

Use it when changing workflow, install behavior, skills, agents, runtime helpers, or provider templates.

## Source Of Truth Order

When two docs disagree, use this order.

```text
[Current Codex Contract]
   |
   +--> templates/codex/AGENTS.md
   +--> templates/codex/skills/*/SKILL.md
   +--> scripts/install_global.py
   |
   v
[Current Explanation]
   |
   +--> README.md
   +--> docs/README.md
   +--> templates/codex/INSTALL.md
   +--> templates/codex/skills/README.md
   +--> templates/codex/agents/README.md
   |
   v
[Support And Memory]
   |
   +--> runtime/README.md
   +--> docs/specs/*
   +--> .everything-automate/decisions/*
   |
   v
[Removed And Inactive Work]
       +--> docs/research/* tombstones
       +--> older docs/specs tombstones
       +--> inactive provider templates
       +--> .everything-automate/plans/*
```

The current user-facing flow is:

```text
$brainstorming
  -> $planning
  -> $execute
  -> $qa
  -> commit
```

Support intake skills are `$issue-capture` and `$issue-pick`.

## Status Words

- `current`
  Active guidance for the Codex-first setup or workflow.
- `support`
  Lower-level runtime, artifact, installer, or decision support.
- `inactive`
  Provider surface with no current implementation.
- `removed`
  Old content was deleted. A small tombstone may remain to avoid broken links.
- `superseded`
  The main guidance has been replaced by a newer source.
- `needs-update`
  Known stale content that should be cleaned before it is used as guidance.

## Drift Checklist

When changing user-facing workflow, setup commands, installed assets, skill roster, agent roster, or project status, check these files:

- `README.md`
- `docs/README.md`
- `docs/docs-maintenance.md`
- `templates/README.md`
- `templates/codex/INSTALL.md`
- `templates/codex/skills/README.md`
- `templates/codex/agents/README.md`
- `.everything-automate/decisions/`

When changing install behavior, also check:

- `scripts/install_global.py`
- `scripts/install_common.py`
- `templates/codex/AGENTS.md`
- `templates/codex/skills/`
- `templates/codex/agents/`

## Useful Checks

Run these targeted searches during docs cleanup.

```bash
rg -n 'Claude Code baseline|will become|will hold|Ralph-ready|4-agent|angel/devil|architect/devil|Current M4|current `M4`|placeholder install guide|draft source-of-truth' README.md AGENTS.md docs templates runtime -g '*.md'
```

For every remaining match, either:

- update the wording, or
- delete the stale body and leave a short tombstone only when the path is still useful.

For installer-facing docs, verify with a temp Codex home:

```bash
python3 scripts/install_global.py setup --provider codex --codex-home /tmp/everything-automate-docs-check-codex-home
python3 scripts/install_global.py doctor --provider codex --codex-home /tmp/everything-automate-docs-check-codex-home
```
